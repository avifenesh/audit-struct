# System Architecture

## Executive Summary

struct-audit follows an **Open Core** architecture with two primary components:
1. **CLI Agent**: Local binary analysis tool (open source)
2. **SaaS Platform**: Historical tracking and collaboration (commercial)

This separation allows standalone usage while providing monetizable team features.

## Component 1: CLI Agent

### Responsibilities

1. **Binary Ingestion**: Load ELF/Mach-O/PE files
2. **DWARF Parsing**: Extract type information
3. **Layout Analysis**: Calculate padding, cache efficiency
4. **Diffing**: Compare two binaries
5. **Reporting**: Output text tables or JSON
6. **CI Gating**: Exit with error code on threshold violation

### CLI Commands

```bash
# Inspect a single binary
struct-audit inspect ./target/release/myapp

# Compare two binaries
struct-audit diff ./main.bin ./feature.bin

# CI mode: fail if thresholds exceeded
struct-audit check ./target/release/myapp --config .struct-audit.yaml

# Upload report to SaaS
struct-audit upload ./target/release/myapp --token $SA_TOKEN

# Explain a specific struct
struct-audit explain ./myapp "my_crate::Order"
```

### Configuration File (.struct-audit.yaml)

```yaml
# .struct-audit.yaml
version: 1

# Structs to monitor (regex patterns)
include:
  - "^my_app::.*"
  - "^core_lib::models::.*"

# Structs to ignore
exclude:
  - "^std::.*"
  - "^alloc::.*"
  - ".*Test.*"

# Size budgets
budgets:
  - pattern: "my_app::Order"
    max_size: 64        # Must fit in one cache line
    max_padding: 8      # Max 8 bytes of padding

  - pattern: "my_app::Tick"
    max_size: 32

# Cache line size (default: 64)
cache_line_size: 64

# Output format
output:
  format: json          # text | json | markdown
  verbose: true

# CI behavior
ci:
  fail_on_new_padding: true
  fail_on_size_increase: true
  baseline_branch: main
```

5. **Visualization**: Dashboards, trend charts, heatmaps
6. **Alerting**: Email/Slack notifications on regressions

### Database Schema

```sql
-- Core entities
CREATE TABLE organizations (
    id UUID PRIMARY KEY,
    name TEXT NOT NULL,
    github_installation_id BIGINT,
    plan TEXT DEFAULT 'free',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE repositories (
    id UUID PRIMARY KEY,
    org_id UUID REFERENCES organizations(id),
    name TEXT NOT NULL,
    full_name TEXT NOT NULL,  -- "owner/repo"
    default_branch TEXT DEFAULT 'main',
    created_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE commits (
    id UUID PRIMARY KEY,
    repo_id UUID REFERENCES repositories(id),
    sha TEXT NOT NULL,
    branch TEXT NOT NULL,
    author TEXT,
    message TEXT,
    timestamp TIMESTAMPTZ,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    UNIQUE(repo_id, sha)
);

-- Struct tracking (time-series optimized)
CREATE TABLE struct_snapshots (
    id UUID PRIMARY KEY,
    commit_id UUID REFERENCES commits(id),
    struct_name TEXT NOT NULL,
    struct_hash TEXT,  -- For deduplication

    -- Metrics
    size_bytes INT NOT NULL,
    padding_bytes INT NOT NULL,
    alignment INT NOT NULL,
    density REAL NOT NULL,
    cache_lines_spanned INT NOT NULL,
    member_count INT NOT NULL,

    -- Full layout (JSONB for flexibility)
    layout JSONB NOT NULL,

    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Indexes for common queries
CREATE INDEX idx_snapshots_commit ON struct_snapshots(commit_id);
CREATE INDEX idx_snapshots_name ON struct_snapshots(struct_name);
CREATE INDEX idx_snapshots_hash ON struct_snapshots(struct_hash);

-- Budgets configuration
CREATE TABLE budgets (
    id UUID PRIMARY KEY,
    repo_id UUID REFERENCES repositories(id),
    pattern TEXT NOT NULL,
    max_size INT,
    max_padding INT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### API Endpoints

```
POST /api/v1/reports
  Upload analysis report from CLI
  Auth: API Token
  Body: JSON report from CLI

GET /api/v1/repos/{repo}/structs
  List all tracked structs
  Query: ?branch=main&limit=100

GET /api/v1/repos/{repo}/structs/{name}/history
  Get historical data for a struct
  Query: ?from=2024-01-01&to=2024-12-31

GET /api/v1/repos/{repo}/diff
  Compare two commits
  Query: ?base=sha1&head=sha2

POST /api/v1/webhooks/github
  GitHub webhook handler
  Auth: GitHub signature verification
```

## Security Considerations

### Data Privacy

1. **Name Hashing Mode**: For sensitive codebases
   ```bash
   struct-audit upload --hash-names --salt $SECRET
   ```
   - Struct names are salted+hashed before upload
   - SaaS sees `struct_a8f3b2c1` instead of `AlphaStrategy`
   - Local CLI can resolve hashes

2. **Self-Hosted Option**: Enterprise customers can run full stack on-premise

### Authentication

- **CLI**: API tokens (organization-scoped)
- **Web**: OAuth via GitHub/GitLab
- **Enterprise**: SAML/SSO support
